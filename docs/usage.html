Basic Usage
Setting up a GDit Project

Setting up a GDit Project

./gdit.exe init some_level_name

To set up a new GDit project, type in the name of the level.

The level you init a GDit from should ideally be the base layout. It can not be changed after initialization.

./gdit.exe get gdit_name

After initialization, you need to run the get command to get the GDit part.
This will place the file to your copies/ folder, from where you can send it to your collab participants.

./gdit.exe C:/Path/To/File.gditl

After sending the file to your collab creators, they can start working on it by running this command.

Replace "C:/Path/To/File.gditl" with the path to the actual file you were sent.
You can automatically copy the path by Shift + Right-Clicking the file in File Explorer and selecting "Copy As Path"
Saving your changes

Saving your changes

./gdit.exe commit gdit_part_name

Committing checks what objects you have added / removed and adds them to a new commit file.

Do not delete commit files.
Merging

Merging

./gdit.exe merge path_to_commit

Merge is not fully finished yet. If you want to use it, you have to merge every commit individually.
